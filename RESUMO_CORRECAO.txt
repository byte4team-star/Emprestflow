  CORREÇÃO CRÍTICA: DATA DE VENCIMENTO GRAVANDO UM DIA ANTES
═══════════════════════════════════════════════════════════════════════════════

📅 PROBLEMA
-----------
Ao criar ou editar contratos, a data de vencimento era gravada com UM DIA ANTERIOR
ao selecionado pelo usuário.

Exemplo: Usuário escolhe 15/04/2026 → Sistema grava 14/04/2026 ❌


🔍 CAUSA
--------
O código usava new Date(year, month, day) que considera o fuso horário LOCAL do
servidor. Com servidores em UTC e conversões de timezone, a data mudava.


✅ SOLUÇÃO
----------
Substituído por Date.UTC(year, month, day) para garantir processamento em UTC
sem conversões indesejadas.


📝 ARQUIVOS CORRIGIDOS
----------------------
/supabase/functions/server/index.tsx

  ✓ Linha ~1151 - Rota POST /contracts (criar contrato)
  ✓ Linha ~1258 - Rota PUT /contracts/:id (editar contrato)  
  ✓ Linha ~2344 - Rota POST /seed (dados de teste)


🚀 DEPLOY NECESSÁRIO
--------------------
A correção está implementada no código mas NÃO está ativa no servidor.

É necessário fazer DEPLOY MANUAL da Edge Function "server" via:
  • Supabase Dashboard, ou
  • Supabase CLI: supabase functions deploy server

Sem o deploy, o problema continuará ocorrendo!
