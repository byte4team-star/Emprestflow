🔧 Teste de Correção: Timezone

Verificando problema de data de vencimento

📅 Teste 1: Problema Original (Timezone Local)
✅ Teste 2: Solução Implementada (UTC)
🔍 O que causava o problema?

O construtor new Date(year, month, day) usa o fuso horário local do servidor. Quando o servidor está em UTC e há conversão de datas, pode ocorrer uma diferença de um dia.

✅ Como foi corrigido?
Usando Date.UTC(year, month, day) para garantir que todas as datas sejam processadas em UTC, evitando conversões indesejadas.
// ❌ ANTES (Problema) const dueDate = new Date(year, month + i, day); const formatted = `${dueDate.getFullYear()}-${dueDate.getMonth() + 1}...`; // ✅ DEPOIS (Correto) const dueDate = new Date(Date.UTC(year, month + i, day)); const formatted = `${dueDate.getUTCFullYear()}-${dueDate.getUTCMonth() + 1}...`;
